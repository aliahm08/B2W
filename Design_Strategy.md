# B2W UX and Design Strategy

Last updated: 2026-03-03

This document defines the active design system for the B2W website.

## 1. UX Principles

- Themeability: default to clean light mode with manual dark-mode toggle.
- Product-first structure: each page behaves like a landing page with hero followed by a demo preview.
- Focused conversion flow: strong CTA hierarchy with minimal navigation distraction.
- Top-right utility CTA: global `Book Intro Call` button lives outside nav links.
- Mission-driven copy: content must map directly to `Mission.md` and avoid non-essential narrative.

## 2. Sana Labs-Inspired Reference (Exact SDL Case Details)

Source: https://www.stockholmdesignlab.se/work/sana-labs

The B2W system should explicitly reflect the Sana case details below:

1. Positioning tone:
- Use a human-centered AI tone inspired by Sana's "Learning made human" framing.
- Keep messaging practical and outcome-oriented, not abstract.

2. Product behavior language:
- Reflect the "personalised, collaborative and engaging" product behavior pattern.
- Prioritize right-time delivery of the right information based on context.

3. Experience architecture:
- Design to "simplify, unify, and amplify" workflows.
- Reduce scattered interactions into one clear operating surface per audience pathway.

4. Visual system direction:
- Use a flexible color system that supports multiple communication contexts.
- Reserve high-contrast accents for optimized digital legibility and CTA hierarchy.
- Use soft studio-like backgrounds, light gradients, and subtle silhouette depth for abstraction.

5. Growth UX loop:
- Build cross-touchpoint journeys (website -> events/content/campaign touchpoints).
- Treat social/content surfaces as referral drivers into key conversion pages.

6. Collaboration model (internal execution):
- Design decisions should be aligned across product, design, marketing, and management inputs.
- The website must operate as both brand system and product demonstration system.

## 3. Information Architecture

- Home introduces the full three-pathway model.
- Individuals page demonstrates messenger-style AI workflow and task execution.
- Enterprises page demonstrates dashboard-style analytics and operational visibility.
- Government page demonstrates secure decisioning and audit-oriented workflow.

## 4. Visual System

- Typeface: Inter for headings and body.
- Density: generous negative space with compact utility controls.
- Surfaces: layered panels with restrained borders and soft depth.
- Motion: subtle fade/scale transitions for orientation, plus intentional mode transitions in demos.
- Demo language: realistic UI previews over abstract icon blocks.

## 5. Header and Navigation

- Brand at left.
- Minimal nav cluster in center.
- Theme toggle + global CTA in top-right action cluster.
- Mobile header uses progressive disclosure via menu button.

## 6. Documentation Sync Rules

For every major UI/content update:

1. Validate against `Mission.md` first.
2. Update this design document for any UX/system change.
3. Add an entry to `CHANGELOG.md`.
